## Grindlab UTM Tracking SOP

**Purpose**  
This SOP explains how Grindlab creates tracked marketing links so Phil can launch campaigns without guessing at naming. The goal is simple: every external campaign link should follow the same structure so reporting by source, medium, campaign, and placement stays clean.

**Core Rule**  
Do not hand-type UTMs directly into live links. Build them in the shared tool, review them, save them to the registry, and then launch.

## When To Use UTMs

Use UTMs on external marketing links such as:

- paid ads
- YouTube links
- partner and affiliate placements
- newsletter and lifecycle emails
- cold email outreach
- social posts, comments, and replies
- blog distribution
- community posts
- PR, sponsorship, and creator placements

Do **not** use UTMs on internal same-domain links. Internal CTAs should use CTA IDs, event tracking, or another internal analytics method so attribution is not overwritten.

## Required Rules

- lowercase only
- hyphens only
- no spaces
- use approved values for `utm_source` and `utm_medium`
- never rename a campaign after it is live
- never reuse an old `utm_id`
- always save the link in the registry before launch
- require `utm_id` for paid, partner, affiliate, and cold outbound links

## Required Parameters

- `utm_source`: where the traffic comes from
- `utm_medium`: what kind of traffic it is
- `utm_campaign`: the business initiative or campaign bucket
- `utm_content`: the specific asset, placement, or variation
- `utm_term`: optional targeting detail such as keyword, subreddit, audience, or segment
- `utm_id`: required for paid, partner, affiliate, and outbound links

## Campaign Core Model

Grindlab now uses a two-part model:

- `Campaign Core`: the reusable campaign definition
- `Link Context`: the per-link execution details

### What goes in the campaign core

Use only the stable details you want to reuse:

- objective
- offer
- audience when it truly matters
- theme when it truly matters
- period generated by the tool

### What changes per link

These are the execution details that may change every time:

- landing page URL
- `utm_source`
- `utm_medium`
- `utm_content`
- `utm_term` when useful
- `utm_id` when required

The main rule: do not rebuild `utm_campaign` from scratch for every link. Reuse the same campaign core for related emails, posts, ads, creator placements, and distribution links.

## Naming Standards

### `utm_campaign`

Format:

```text
[objective]-[offer]-[audience]-[theme]-[period]
```

Example:

```text
acq-free-tool-cash-players-ev-q2-2026
```

Only `objective`, `offer`, and the auto-generated `period` are expected most of the time. `audience` and `theme` are optional and should only be used when they create reporting value.

### `utm_content`

Format:

```text
[asset-type]-[placement-or-hook]-[variant]
```

Examples:

```text
video-description-main-v1
image-tilt-hook-v2
email-seq1-step2-v1
creator-dougpolk-description-v1
```

## Channel Conventions

### Google Ads

- `utm_source=google`
- `utm_medium=cpc`

Example:

```text
?utm_source=google&utm_medium=cpc&utm_campaign=acq-free-tool-cash-players-ev-q2-2026&utm_content=image-problem-aware-v1&utm_id=google-cpc-q2-2026-01
```

### Paid YouTube

- `utm_source=youtube`
- `utm_medium=paid_video`

### Owned YouTube Links

- `utm_source=youtube`
- `utm_medium=organic_video`

Use separate placements such as:

- `description-main-cta`
- `pinned-comment`
- `end-screen`
- `channel-banner`
- `about-page`

Example:

```text
?utm_source=youtube&utm_medium=organic_video&utm_campaign=acq-free-tool-postflop-study-q2-2026&utm_content=description-main-cta-v1
```

### Other YouTube Channels

- `utm_source=youtube`
- `utm_medium=partner` for creator or sponsorship traffic
- `utm_medium=affiliate` for affiliate relationships

Example:

```text
?utm_source=youtube&utm_medium=partner&utm_campaign=acq-free-tool-cash-players-q2-2026&utm_content=creator-dougpolk-description-v1&utm_id=yt-partner-01
```

### Reddit And Meta Paid Traffic

- `utm_medium=paid_social`
- use the platform name as `utm_source`

### Cold Email

- use sending platform as `utm_source`
- use `cold_email` as `utm_medium`

### Founder Posts And Organic Social

- use platform name as `utm_source`
- use `organic_social` as `utm_medium`

### Social Comments And Community Promotion

- use platform name as `utm_source`
- use `organic_comment` as `utm_medium`

### Blog Distribution

Use UTMs on the external link that sends traffic to the blog post. Do not use UTMs on same-domain CTA links from the blog into the site.

If the blog and app live on different domains, preserve the original UTMs and use CTA metadata for internal CTA analysis when needed.

### Internal CTA Tracking

Do not add new UTMs to same-domain links such as a CTA from a Grindlab blog post to another Grindlab page. Use internal CTA metadata instead, for example:

```text
?cta_id=blog-inline-demo&article_slug=memory-study-system
```

### Kit-Wrapped Links

If Kit wraps the public destination inside a redirect link, still treat the real destination and its UTMs as the source of truth. The builder should generate the final tracked destination URL first, then that URL can be placed into Kit.

### Reusing One Campaign Core Across Many Links

Example reusable campaign core:

```text
acq-free-tool-cash-players-q2-2026
```

Possible links built from that one core:

- newsletter main CTA
- owned YouTube description CTA
- partner YouTube description CTA
- paid Reddit ad
- founder LinkedIn post

## Phil Workflow

1. Pick the landing page.
2. Reuse an existing campaign core or create a new one.
3. Pick the link type, source, and medium.
4. Fill in the content helper fields.
5. Check whether `utm_id` is required.
6. Copy the generated URL.
7. Save the link to the registry.
8. Launch only after QA is done.

## QA Checklist

Before any link goes live, confirm:

- the URL opens correctly
- the landing page is correct
- `utm_source` is approved
- `utm_medium` is approved
- `utm_campaign` matches the initiative
- `utm_content` clearly identifies the asset or placement
- `utm_id` is present when required
- the link has been saved in the registry
- a short link exists if the destination will be shown publicly

## Ownership

- Phil or marketing execution creates the link in the builder
- Phil or reviewer confirms the QA checklist
- paid, partner, affiliate, and outbound links should be reviewed before launch
